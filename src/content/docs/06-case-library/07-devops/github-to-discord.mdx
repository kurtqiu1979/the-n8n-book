> **难度** 🟢 简单 · **耗时** ~15 分钟 · **核心节点** GitHub Trigger × Switch × Discord

开源项目或小团队 Discord 服务器最常见的诉求——让 GitHub 的关键事件自动推到对应频道。PR 到 #dev-prs，Issue 到 #dev-issues，Release 到 #announcements。**3 个分支、3 类讯息一目了然**。

Route Event (Switch):
  Output 0: action = "opened"      (PR)
  Output 1: action = "opened"      (Issue)
  Output 2: action = "published"   (Release)

  Discord #dev-prs

<TipCallout type="warn" title="坑 · 不同 event 的 payload 结构不一样">
  
  `pull_request` 事件用 `$json.pull_request.title`，`issues` 用 `$json.issue.title`，`release` 用 `$json.release.tag_name`。**节点 webhook payload 结构与事件类型强相关**——参考 GitHub Webhook 文档。
  
</TipCallout>

<TipCallout type="quest" title="扩展挑战">
  
  - 加 IF：只在 PR 包含 "[breaking]" 时 @通知整个频道
  - 用 AI 节点把长 PR 描述自动总结成 3 句话
  - 监听多个 repo：动态用 SplitOut + 一个 Set 节点配置 repo 列表
  
</TipCallout>
