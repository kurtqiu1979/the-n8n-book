> **难度** 🟢 简单 · **耗时** ~15 分钟 · **核心节点** Schedule × HTTP × IF × Discord

UptimeRobot 之类的服务每月几十刀。你的小项目其实只要一个 5 节点 n8n workflow 就能搞定——每 5 分钟 ping 一次健康检查端点，挂了立刻 Discord 通知，所有结果落 Airtable 做 SLA 统计。

## 📋 关键节点

<TerminalCell filename="key-config.txt" runTag="⚙">
{`Every 5 min (Schedule):
  Minutes: 5

Health Check (HTTP):
  URL: https://your-site.com/health
  Options:
    Timeout: 10000
    Full Response: true   ← 才能拿到 statusCode
    Ignore Response Code: true   ← 5xx 也不抛错

Is Down? (IF):
  $json.statusCode != 200

Discord Alert (True 分支):
  Channel: #ops-alerts
  Text: 🔴 Site DOWN · status {{ statusCode }}

Log to Airtable (双分支都走):
  url, status, duration_ms, checked_at`}
</TerminalCell>

<TipCallout type="warn" title="坑 · 告警风暴">
  
  站点挂 1 小时 = 12 次连续告警，瞬间淹没频道。**加防抖**：用 Static Data 记 last_alert_time，连续告警的间隔 < 30 分钟就跳过。
  
</TipCallout>

<TipCallout type="quest" title="扩展挑战">
  
  - 监控多个 URL：Set 数组 → SplitInBatches → 并行检查
  - 加 IF：响应慢（> 3s）也告警，不只 down
  - SLA 看板：从 Airtable 数据做月度图表
  
</TipCallout>
