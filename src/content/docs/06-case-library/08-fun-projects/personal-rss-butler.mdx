> **难度** 🟡 中等 · **耗时** ~25 分钟 · **核心节点** RSS × Aggregate × AI Curator

很多人有"RSS 订阅有几十个，但 RSS 阅读器从不打开"的悖论。这个 workflow 替你**每天扫一遍**，AI 按主题分类 + 挑出 5 篇精华、写摘要，Telegram 推一份 Markdown 给你——**通勤路上 5 分钟读完今日值得看的**。

## 📋 关键节点

<TerminalCell filename="key-config.txt" runTag="⚙">
{`Set Feed URLs:
  feeds: "https://news.ycombinator.com/rss,
          https://blog.n8n.io/feed,
          https://www.anthropic.com/news/rss.xml"

Split Feeds:
  Field: feeds, Delimiter: ","
  → 每条 feed 一个 item

Read RSS:
  URL: ={{ $json.feeds }}
  Ignore Older Than: 86400 (秒) ← 只要最近 24 小时

Aggregate:
  Fields: [title, link]
  → 把多条 feed 的 item 聚合成一个数组

AI Curator:
  System: 按主题分类（AI / Tech / Product / Other）
          挑 5 篇最值得看的写 200 字摘要
          中文 Markdown，带链接

Send to Telegram:
  parseMode: Markdown
  ← 链接才能渲染`}
</TerminalCell>

<TipCallout type="warn" title="坑 · RSS 源停服">
  
  长期跑下来肯定有几个 RSS 源会挂掉或改 URL。**每个 Read RSS 开 Continue On Fail**，一个挂了不影响其他源。
  
</TipCallout>
