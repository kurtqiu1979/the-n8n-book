n8n 内置 **500+ 应用节点**——从 Slack、Notion 到 OpenAI、Shopify。这些"现成节点"是 n8n 比自己写 HTTP 调用更高级的地方。这一节带你按类别速览，让你**遇到一个需求能秒判断"该用哪个节点"**。

## 按类别速览

### 💬 沟通与协作（最高频）
**Slack · Discord · Telegram · Microsoft Teams · Mattermost · Email Send · Gmail**

发消息、收消息、上传文件、管理频道。Slack 节点是 n8n 用户最高频的之一。

### 📋 任务与协作工具
**Notion · Airtable · Trello · ClickUp · Asana · Linear · Jira**

读/写数据库行、新建任务、更新状态。Notion 节点尤其热门。

### 🛒 电商与支付
**Shopify · Stripe · PayPal · WooCommerce · Square**

订单事件、退款触发、库存同步。

### ☁️ 云存储与文件
**Google Drive · Dropbox · OneDrive · AWS S3 · FTP / SFTP**

上传/下载/移动文件、列目录、文件元信息。

### 📊 数据库
**PostgreSQL · MySQL · MongoDB · Redis · Supabase · Snowflake · BigQuery**

直接执行 SQL、增删改查、批量插入。

### 📧 邮件营销 & CRM
**HubSpot · Salesforce · Mailchimp · ActiveCampaign · Pipedrive · Intercom**

联系人 CRUD、列表订阅、邮件发送。

### 🤖 AI / LLM（增长最快的类别）

### 🛠 开发者工具
**GitHub · GitLab · Bitbucket · Jenkins · GraphQL · MQTT · WebSocket**

代码事件、CI/CD 触发、订阅消息。

### 🌐 内容平台
**WordPress · Webflow · Ghost · YouTube · Twitter / X · Reddit · Bluesky**

发布文章、抓取数据、社交互动。

## 应用节点 vs HTTP Request：选哪个？

每个 SaaS 应用 n8n 都有节点了，**为什么还要 HTTP Request 节点？** 因为：

| 维度 | 应用节点 | HTTP Request |
|---|---|---|
| **开发速度** | 快（UI 表单填一填） | 慢（要查 API 文档） |
| **覆盖度** | 覆盖该应用的常用 API | 覆盖该应用的**所有** API |
| **新功能跟进** | n8n 团队更新 | 应用一发新 API 你就能用 |
| **认证** | 一次配 Credentials | 每个 workflow 都要手动配 |

**心法**：**有现成应用节点就用现成的，没有或者它不支持的功能再 fallback 到 HTTP Request**。

<TerminalCell filename="when-to-fallback.txt" runTag="🎯">
{`场景 A · "我要发个 Slack 消息到 #general"
  → Slack 节点（一行配完）

场景 C · "我要用一个 n8n 没节点的小众工具（如 Linear API）"

## 应用节点的"操作"二维选择

打开任一应用节点，会有两个下拉：

- **Resource** —— 操作的对象类型（如 Notion 的 `database / page / block`）
- **Operation** —— 对该对象做什么（`get / create / update / delete / list / search`）

<TerminalCell filename="notion-resource-operation.txt" runTag="📦">
{`Notion 节点的 Resource × Operation 组合：

Resource: Database
  Operations: get, getMany, getDatabasePages, search

Resource: Page
  Operations: get, getMany, create, update, archive

Resource: Block
  Operations: getMany, append, update, delete

Resource: User
  Operations: get, getMany`}
</TerminalCell>

这套 Resource × Operation 设计在 50+ 应用节点里都一致——**学会一个，等于学会一类**。

<TipCallout type="warn" title='新手最大的坑 · "节点报错说权限不够"'>
  90% 是 Credentials 配置问题——OAuth scope 没给足、API Key 没勾对应权限。下一节[Credentials 凭证体系](../credentials/) 详讲。
</TipCallout>

## 找不到节点怎么办？

1. **检查搜索关键词** —— n8n 节点库搜索匹配比较松，比如搜 "Notion" 会出来 Notion 主节点 + Notion Trigger，搜 "Sheets" 出 Google Sheets。
2. **用 HTTP Request 节点** —— 实在没有，HTTP Request + 该应用的 API 文档，照样能调。
3. **社区节点** —— n8n 支持安装第三方社区节点（自托管版才可），https://www.npmjs.com/search?q=n8n-nodes- 上有 300+ 社区维护的节点。
4. **自定义节点** —— 终极方案：自己写一个节点（本书不展开，超出"零基础"范围）。

<TipCallout type="quest" title="扩展挑战">
  挑你最熟悉的一个 SaaS 应用，打开 n8n 看它的节点里有哪些 Resource × Operation。然后对照该应用的官方 API 文档，看节点覆盖了多少比例的 API。
</TipCallout>

## 本节要点回顾

- 500+ 应用节点按类别覆盖主流 SaaS 工具

- 每个应用节点的核心是 **Resource × Operation** 二维选择
- 90% 的"节点报错"来自 Credentials 配置问题
