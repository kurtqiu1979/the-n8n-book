Credentials（凭证）是 n8n 节点"代表你"去调用第三方 API 的身份证。**90% 的"节点报错"都跟凭证配错有关**——这一节专治这个。

## 凭证 = 加密保管的身份信息

每个 Credentials 记录包含：
- **类型**（API Key / OAuth2 / Header Auth / Basic Auth …）
- **字段**（具体的 key、token、URL 等，依类型而定）
- **加密存储**（用 n8n 的 encryptionKey 加密后落数据库，前端永远只显示 `********`）

凭证是 workflow 之外的独立对象——**一份凭证可以被多个 workflow 共用**。

## 三种主流凭证类型

### 类型 1 · API Key（最简单）

适合：内部工具、命令行风格的 API（OpenAI、Anthropic、Stripe Secret Key）。

<TerminalCell filename="api-key.txt" runTag="🔑">
{`Credential Type: <某节点的> API
Fields:
  API Key:  sk-proj-xxxxxxxxxxxxxxx

节点使用时会自动注入：
  Header: Authorization: Bearer sk-proj-xxxxxxxxxxxxxxx`}
</TerminalCell>

**何时找它？** 应用的"Developer Settings → API Keys"页面里生成。

### 类型 2 · OAuth2（最常见但最复杂）

适合：Google / Microsoft / Notion / Slack / GitHub 这类要求"代表用户操作"的服务。

OAuth2 流程：

<TerminalCell filename="oauth2-flow.txt" runTag="🔐">
{`1. 在 n8n 新建 Credential，类型选 "Notion OAuth2 API"
2. n8n 显示一个 redirect URL，复制
3. 去 Notion → My integrations → 你的 OAuth integration → 把 redirect URL 粘进去
4. 回到 n8n 点 "Connect my account"
5. 浏览器弹出 Notion 授权页 → 选择允许的页面
6. 重定向回 n8n → 凭证保存
7. 节点的 Resource 列表里就能选到你授权的页面/数据库了`}
</TerminalCell>

<TipCallout type="warn" title="OAuth2 三个常见坑">
  
  1. **Redirect URL 没填对** —— 复制粘贴时千万别漏字符，结尾的 `/` 不能错
  2. **Scope 不够** —— OAuth 应用在配置时要勾的权限范围（如 `read:user`），少勾了节点跑不通
  3. **Token 过期** —— 长期不用 OAuth 凭证会失效，重新点 "Connect my account" 续上即可
  
</TipCallout>

### 类型 3 · Header Auth / Basic Auth（fallback）

适合：HTTP Request 节点调一些不在内置列表里的 API。

<TerminalCell filename="header-auth.txt" runTag="🔑">
{`Header Auth:
  Name:   Authorization
  Value:  Bearer your-token-here

Basic Auth:
  User:     admin
  Password: secret

Custom Auth（高级）：
  完整自定义请求拼装 —— 用于复杂签名场景`}
</TerminalCell>

## 在节点里使用凭证

每个支持凭证的节点，参数面板顶部有一个 **Credentials** 下拉框：

<TerminalCell filename="select-credential.txt" runTag="🔌">
{`Slack 节点参数面板：
  ▶ Credential to connect with:    [Slack account (kurt) ▼]
                                   [+ Create new credential ]
  ▶ Resource:    Message
  ▶ Operation:   Post
  ▶ Channel:     #general
  ...`}
</TerminalCell>

**一个节点只能选一份凭证**。如果你需要"用账号 A 发到 channel X，再用账号 B 发到 channel Y"，要放两个节点各自选凭证。

## 凭证共享（团队场景）

n8n Cloud 团队版和自托管支持**凭证共享**：

- 凭证的 **Sharing** tab → 添加用户/组
- 被共享的用户可以**使用**这份凭证（节点选得到），但**看不到**敏感字段（永远是 `********`）
- 创建者可以随时撤销共享

<TipCallout type="tip" title="备份 encryptionKey">
  n8n 用 `~/.n8n/config` 里的 `encryptionKey` 加密所有凭证。**这个 key 丢了，所有凭证都得重新填一遍**。Docker 部署务必备份这个文件，做迁移时一并带走。
</TipCallout>

## 凭证测试

新建凭证后，**第一件事是点 "Test step" 验证连通性**。绝大多数凭证页面右下有这个按钮。如果 Test 失败，节点也跑不通——先解决再说。

<TipCallout type="quest" title="扩展挑战">
  装好你的第一个 n8n 后，配三种凭证：① 一个 API Key 类（如 OpenAI）② 一个 OAuth2 类（如 Google Sheets）③ 一个 Header Auth（用于任意需要 Bearer token 的 API）。亲手过一遍流程，OAuth 那个坑最深也最值得踩。
</TipCallout>

## 本节要点回顾

- 凭证 = 加密保管的身份信息，独立于 workflow
- 三种主流：API Key（简单）/ OAuth2（最常见）/ Header Auth（fallback）
- OAuth2 三大坑：redirect URL / scope / token 过期
- 节点参数面板顶部选凭证；一个节点一份凭证
- 自托管务必备份 `encryptionKey`
