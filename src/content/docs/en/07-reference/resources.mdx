## 🔧 Advanced tooling

### Self-hosting depth

- **PostgreSQL tuning** — required as execution data grows
- **Redis Queue Mode** — single instance → multi-worker horizontal scale
- **Reverse Proxy** — Nginx / Caddy / Traefik for HTTPS
- **Backup strategy** — encryption key + execution data + workflow definitions

## 🎓 Advanced paths (by goal)

### "I want to build a product on n8n"
1. Learn **Queue Mode** + multi-worker deployment
2. Learn **RBAC** / multi-tenancy / team management
3. Learn **error handling + monitoring** for production
4. Read [restyler/awesome-n8n](https://github.com/restyler/awesome-n8n) "production patterns"

### "I want to build AI Agents with n8n"
1. Do **every challenge** in Part 5
2. Read LangChain JS docs cover-to-cover
3. Play with Pinecone / Qdrant, build 5+ RAG demos
4. Join LangChain Discord / r/LocalLLaMA

### "I want to develop custom n8n nodes"
1. Learn TypeScript (if not yet)
2. Read [n8n-nodes-starter](https://github.com/n8n-io/n8n-nodes-starter)
3. Read source of a community node or two (e.g. `n8n-nodes-tools`)
4. Publish to npm

### "I want to migrate from Make / Zapier"
1. Check [docs.n8n.io migration guides](https://docs.n8n.io)
2. Recreate existing Zaps using Part 6 cases as templates
3. Switch to Cloud (less ops) or Docker (cheaper at scale)
