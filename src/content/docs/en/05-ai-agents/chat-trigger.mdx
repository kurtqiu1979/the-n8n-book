So far your Agent only chats inside the n8n editor. Last section: **ship it** — into Slack, Telegram, WhatsApp, or embedded in your own website.

## Four entry-point options

| Approach | Node | For |
|---|---|---|
| **n8n Chat UI** | Chat Trigger | Internal use, prototype demos |
| **Embed on site** | Chat Trigger + frontend SDK | Customer-facing bot |
| **Slack / Discord / Teams** | App Trigger (Slack Trigger etc.) | Team-internal bot |
| **Telegram / WhatsApp** | Telegram Trigger / WhatsApp Trigger | Personal assistant |

## Option 1 · n8n's built-in Chat UI

Chat Trigger generates a **public chat page URL**. Open it, chat:

<TerminalCell filename="chat-trigger-config.txt" runTag="💬">
{`Chat Trigger parameters:
  Make Chat Publicly Available:  ON  (required)
  Authentication:                None / Basic / n8n User Auth
  Initial Messages:              "Hi! Ask me about the handbook."
  Allowed Origins (CORS):        *  (embeddable anywhere)
  Title:                         "AI Assistant"

After activation, the node shows two URLs:
  - Chat URL (open directly, get a chat interface)
  - Webhook URL (used by frontend SDK)`}
</TerminalCell>

Send the Chat URL to colleagues — zero frontend code.

## Option 2 · Embed in your site

n8n provides `@n8n/chat`, a few lines embed it anywhere:

<TerminalCell filename="embed-chat.txt" runTag="🌐">
{`<!-- your site HTML -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/@n8n/chat/dist/style.css" />
<script type="module">
  import { createChat } from 'https://cdn.jsdelivr.net/npm/@n8n/chat/dist/chat.bundle.es.js';

  createChat({
    webhookUrl: 'https://n8n.example.com/webhook/<chat-trigger-id>',
    mode: 'window',        // or 'fullscreen'
    initialMessages: ['Hi!'],
  });
</script>

Effect: chat bubble at bottom-right of your page; click to open and chat.`}
</TerminalCell>

## Option 3 · Slack / Discord

Swap Chat Trigger for **Slack Trigger**, output goes to Slack send-message node:

<TerminalCell filename="slack-bot.txt" runTag="💼">
{`[Slack Trigger]                           listen for @AI-Bot mentions
   Trigger Event:  App Mention
   ↓
[AI Agent]
   Chat Model:  OpenAI
   Memory:      Postgres Chat Memory
                Session Key: ={{ $json.user }}    Slack user ID as session key
   Tool:        Vector Store Tool / HTTP Tool / etc.
   Prompt:      ={{ $json.text }}                  message after stripping @
   ↓
[Slack - Send Message]
   Channel:  ={{ $('Slack Trigger').item.json.channel }}    back to origin channel
   Text:     ={{ $json.output }}
   Thread:   ={{ $('Slack Trigger').item.json.ts }}         reply in thread`}
</TerminalCell>

Now teammates can `@AI-Bot help me find...` in any Slack channel and get a threaded reply.

## Streaming output

Default: Agent waits for full response, returns once. To get the ChatGPT-style **character-by-character** stream:

- **Chat Trigger's built-in UI supports streaming** — enable "Enable Streaming" on Agent
- **Embedded chat (@n8n/chat)** same
- **Slack / Telegram** don't support streaming (API limitations); only complete responses

## Handling user file uploads

Chat Trigger supports image/file uploads:

<TerminalCell filename="file-handling.txt" runTag="📎">
{`Chat Trigger config:
  Allow File Uploads: ON
  Max File Size:      10 MB
  Allowed Mime Types: image/*, application/pdf

After upload, workflow receives:
  $json.chatInput      text part
  $json.files          uploaded files array

Downstream:
  - Image → Vision node (OpenAI gpt-4o vision)
  - PDF → §5.5 RAG ingest flow`}
</TerminalCell>

## Real: connect §5.5 RAG assistant to Telegram

<TerminalCell filename="rag-on-telegram.txt" runTag="🎯">
{`Workflow: "RAG Telegram Bot"

[Telegram Trigger]
   ↓
[AI Agent]
   Chat Model:  OpenAI gpt-4o-mini
   Memory:      Postgres Chat Memory (Session Key = chat.id)
   Tool:        Vector Store Tool (pinecone "my-knowledge-base")
   Prompt:      ={{ $json.message.text }}
   ↓
[Telegram - Send Message]
   Chat ID:  ={{ $('Telegram Trigger').item.json.message.chat.id }}
   Text:     ={{ $json.output }}

Effect: chat with the bot on Telegram, it answers from the handbook you ingested in §5.5,
with cross-device persistent memory (Postgres).`}
</TerminalCell>

<TipCallout type="warn" title="Three production security considerations">
  
  1. **Public Chat URL is exposed**: anyone can chat → burns your API quota. Before going live, add Authentication (Basic Auth / n8n User Auth).
  2. **Abuse protection**: rate-limit (N calls/min per user) — Set node + Postgres logging.
  3. **PII filtering**: users may type phone numbers, IDs — sanitize with a Code node before passing to LLM.
  
</TipCallout>

## Section recap (and Part 5 recap)

- 4 entry points: n8n Chat UI / embed on site / Slack-Discord / Telegram-WhatsApp
- Slack & Telegram integration: Trigger → AI Agent → Send Message
- Memory Session Key uses platform user ID (chat.id / user.id)
- Streaming: Chat UI / embed support; IM platforms don't
- File uploads: Chat Trigger supports natively; downstream Vision / RAG ingest
- **Before going live**: Auth, rate limit, PII filtering
