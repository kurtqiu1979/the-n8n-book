> **Difficulty** 🟡 Medium · **Time** ~25 min · **Core nodes** S3 List × Code logic × Date check

"Backups are running" — until the day you lose data and discover backups have been broken for 6 months. This workflow checks daily for today's backup file in S3; missing = alert. Fridays send a weekly summary confirming this week's backups are healthy.

## 📋 Key configs

<TerminalCell filename="key-config.txt" runTag="⚙">
{`Daily 9AM Check (Schedule):
  Cron: 0 9 * * *

List S3 Backups:
  Operation: List
  Bucket:    company-backups
  Prefix:    daily/

Check Today (Code):
  const today = new Date().toISOString().slice(0,10);
  const found = $input.all().some(i =>
    (i.json.key || '').includes(today)
  );
  return [{ json: { found, today, totalFiles: $input.all().length } }];

Backup Missing? (IF):
  found == false → Slack Alert
  found == true  → Is Friday? → weekly report

Is Friday? (IF):
  $now.weekday == 5  ← Luxon: 1=Mon, 5=Fri

Friday Report:
  Email summary of the week's backup health`}
</TerminalCell>

<TipCallout type="warn" title="Trap · 'Backup exists' ≠ 'Backup works'">
  
  This workflow only checks "today's filename exists," not file integrity. Production should add:
  - File size check (anomalously small = possibly empty backup)
  - Sample download + MD5 verification
  - Periodic actual "restore drills"
  
</TipCallout>

<TipCallout type="quest" title="Extensions">
  
  - Monitor multiple backup buckets (database / files / user data)
  - Vector Store: index historical backup metadata, AI detects "backup size dropped 50%" anomalies
  - Monthly restore drill: auto-restore one backup in a sandbox env to verify usability
  
</TipCallout>
