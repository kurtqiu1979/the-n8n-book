<span class="editorial-label">Part 6 · Case 6.1 · 📣 Marketing & CRM</span>

> **Difficulty** 🟡 Medium · **Time** ~30 min · **Core nodes** Webhook × Clearbit × AI Score × HubSpot

The marketing "Get a demo" form only collected an email. This backend workflow auto-enriches via Clearbit (company, role, size), then AI scores (0-100) with a one-line review → writes to HubSpot. **Every lead the sales team sees is already enriched.**

## 📋 Key configs

<TerminalCell filename="key-config.txt" runTag="⚙">
{`Lead Webhook (Webhook):
  Path:        /new-lead
  HTTP Method: POST
  Body:        { email, ... }

Enrich via Clearbit (HTTP):
  URL: =https://person.clearbit.com/v2/people/find?email={{ $json.body.email }}
  Auth: Header Auth (Clearbit API key)

AI Lead Scoring:
  System: Score 0-100 based on Clearbit data + one-line rationale.
          Consider: company size, industry, role seniority.
          Output JSON: { score, reason }
  Response Format: json_object

Create HubSpot Contact:
  Email:    ={{ $('Lead Webhook').body.email }}
  Company:  ={{ $('Clearbit').employment.name }}
  LeadScore: ={{ $json.score }}`}
</TerminalCell>

<TipCallout type="warn" title="Trap · Clearbit 404, continue anyway">
  
  Personal emails are often not in Clearbit; the HTTP node gets 404. **Continue On Fail must be on**, then a downstream IF checks `$json.error` — write to HubSpot regardless, just without enrichment.
  
</TipCallout>

<TipCallout type="quest" title="Extensions">
  
  - High-score leads (>70) → IF + immediate Slack/DingTalk ping to sales
  - Dual enrichment: Clearbit + Apollo/LinkedIn for better hit rate
  - SplitInBatches to rate-limit Clearbit calls
  
</TipCallout>
