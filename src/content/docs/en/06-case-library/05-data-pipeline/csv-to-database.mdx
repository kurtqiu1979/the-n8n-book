<span class="editorial-label">Part 6 · Case 5.1 · 🔄 Data Pipeline</span>

> **Difficulty** 🟡 Medium · **Time** ~25 min · **Core nodes** GDrive Trigger × Extract × IF × Postgres

Ops dumps a 5000-row CSV: "import this." Direct import fails on half (dirty data); manual cleaning is tedious. This workflow watches a Drive folder, **parses new CSVs, validates per row, inserts good rows, emails bad ones for fixing**.

## 📋 Key configs

<TerminalCell filename="key-config.txt" runTag="⚙">
{`GDrive New CSV (Trigger):
  Event:    fileCreated
  Folder:   <watched Drive folder>
  File Type: csv

Extract from File:
  Operation: CSV
  → one item per row

Validate Row (IF):
  Email regex: ^[\\w.-]+@[\\w.-]+\\.\\w+$
  → True branch = insert; False branch = alert

Insert Postgres:
  Operation: Insert
  Table:     customers
  Columns:   email, name, signup_date

Alert Bad Rows:
  Subject: Invalid rows in CSV import
  Text:    ={{ JSON.stringify($json) }}`}
</TerminalCell>

<TipCallout type="warn" title="Trap · Re-import duplicates">
  
  Re-uploading the same file fires again. Either pre-check with SELECT or use Postgres `ON CONFLICT DO NOTHING` (UPSERT mode).
  
</TipCallout>

<TipCallout type="quest" title="Extensions">
  
  - AI Vision node for image-style CSVs (screenshots)
  - Bad rows → separate "to-fix" table
  - SplitInBatches to rate-limit INSERTs, prevent DB overload
  
</TipCallout>
