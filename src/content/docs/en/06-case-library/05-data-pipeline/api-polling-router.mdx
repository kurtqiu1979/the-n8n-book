<span class="editorial-label">Part 6 · Case 5.2 · 🔄 Data Pipeline</span>

> **Difficulty** 🟡 Medium · **Time** ~25 min · **Core nodes** Schedule × Split Out × Switch

Legacy third-party system **without webhooks**? Polling is your only option. This workflow demonstrates the standard "poll every 5 min + route events" pattern — fetch a batch of new events, route by `type` to different Slack channels.

## 📋 Key configs

<TerminalCell filename="key-config.txt" runTag="⚙">
{`Every 5 min (Schedule):
  Mode: Every N Minutes
  Minutes: 5

Fetch Events:
  GET https://api.example.com/events?since=last_poll
  Returns { events: [{ type, payload }, ...] }

Split Events:
  Field: events
  → array → N items

Route by Event Type (Switch):
  Output 0: type = "user.created"
  Output 1: type = "order.placed"
  Output 2: type = "subscription.cancelled"

Three Slack nodes connect to the three Switch outputs`}
</TerminalCell>
