Credentials are how nodes act "on your behalf" when calling third-party APIs. **90% of "node errors" trace back to credential misconfig** — this section fixes that.

## Credentials = encrypted identity vault

Each Credential record contains:
- **Type** (API Key / OAuth2 / Header Auth / Basic Auth / …)
- **Fields** (keys, tokens, URLs — varies by type)
- **Encrypted at rest** (your n8n encryption key encrypts before DB write; UI always shows `********`)

Credentials are independent of workflows — **one credential, many workflows**.

## Three common types

### Type 1 · API Key (simplest)

For internal tools and "CLI-style" APIs (OpenAI, Anthropic, Stripe Secret Key).

<TerminalCell filename="api-key.txt" runTag="🔑">
{`Credential Type: <Node>API
Fields:
  API Key:  sk-proj-xxxxxxxxxxxxxxx

The node auto-injects:
  Header: Authorization: Bearer sk-proj-xxxxxxxxxxxxxxx`}
</TerminalCell>

**Where to get it:** the app's "Developer Settings → API Keys" page.

### Type 2 · OAuth2 (common, complex)

For Google / Microsoft / Notion / Slack / GitHub — services that require "act on behalf of user."

<TerminalCell filename="oauth2-flow.txt" runTag="🔐">
{`1. Create credential in n8n; type "Notion OAuth2 API"
2. n8n shows a redirect URL — copy it
3. In Notion → My integrations → your OAuth integration → paste the redirect URL
4. Back in n8n → click "Connect my account"
5. Browser opens Notion's auth page → grant access
6. Redirect lands back in n8n → credential saved
7. Node's Resource list now shows your authorized pages/DBs`}
</TerminalCell>

<TipCallout type="warn" title="Three OAuth2 traps">
  
  1. **Redirect URL mismatched** — copy-paste exactly; trailing `/` matters
  2. **Insufficient scopes** — set them when configuring the OAuth app; not enough = node fails
  3. **Expired tokens** — long-idle OAuth credentials expire; click "Connect my account" again
  
</TipCallout>

### Type 3 · Header Auth / Basic Auth (fallback)

For HTTP Request node when calling APIs without dedicated nodes.

<TerminalCell filename="header-auth.txt" runTag="🔑">
{`Header Auth:
  Name:   Authorization
  Value:  Bearer your-token-here

Basic Auth:
  User:     admin
  Password: secret

Custom Auth (advanced):
  Fully custom request assembly for tricky signing schemes`}
</TerminalCell>

## Using credentials in nodes

Every credential-aware node has a **Credentials** dropdown at the top of the params panel:

<TerminalCell filename="select-credential.txt" runTag="🔌">
{`Slack node params:
  ▶ Credential to connect with:    [Slack account (kurt) ▼]
                                   [+ Create new credential ]
  ▶ Resource:    Message
  ▶ Operation:   Post
  ▶ Channel:     #general
  ...`}
</TerminalCell>

**One node = one credential.** If you need "account A → channel X, account B → channel Y," use two nodes.

## Credential sharing (team)

n8n Cloud Team and self-host support **credential sharing**:

- Credential → Sharing tab → add users/groups
- Recipients can **use** it (select in nodes), but **cannot see** sensitive fields (always `********`)
- Creator can revoke anytime

<TipCallout type="tip" title="Back up encryptionKey">
  Credentials are encrypted with the key in `~/.n8n/config`. **Lose this file and every credential must be re-entered.** Back it up for any self-hosted deploy.
</TipCallout>

## Test before relying

Most credential forms have a **"Test step"** button. Click it before saving — if it fails, the node won't work either. Diagnose first.

<TipCallout type="quest" title="Extension challenge">
  Set up three credentials: ① API Key (OpenAI) ② OAuth2 (Google Sheets) ③ Header Auth (any service needing Bearer token). OAuth2 is the deepest pit — best worth practicing.
</TipCallout>

## Section recap

- Credentials = encrypted identity, independent of workflows
- Three common types: API Key (simple) / OAuth2 (most common) / Header Auth (fallback)
- OAuth2 traps: redirect URL / scope / token expiry
- One node, one credential
- Self-hosted: back up `encryptionKey`
